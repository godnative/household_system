# 户籍管理系统 - 启动指南

## 1. 开发环境启动

### 1.1 环境要求
- Python 3.7 或更高版本
- pip 包管理器

### 1.2 依赖安装
1. 打开命令行终端，进入项目根目录
2. 执行以下命令安装依赖：
   ```bash
   pip install -r requirements.txt
   ```

### 1.3 数据库初始化
系统首次启动时会自动初始化数据库，包括：
- 创建所有必要的表结构
- 初始化默认角色和权限
- 创建默认超级管理员账号
- 创建默认村数据

### 1.4 启动系统
在项目根目录执行以下命令：
```bash
python -m src.app
```

## 2. 打包后可执行文件运行

### 2.1 可执行文件位置
打包后的可执行文件位于 `dist` 目录中：
- Windows: `dist/app.exe`

### 2.2 运行方法
1. 找到 `dist/app.exe` 文件
2. 双击该文件即可启动系统

### 2.3 注意事项
- 可执行文件是独立的，不需要安装Python环境
- 首次运行时会自动初始化数据库
- 数据库文件会保存在与可执行文件同目录的 `data` 文件夹中

## 3. 数据库初始化

### 3.1 初始化过程
- 系统首次启动时会自动检查数据库是否存在
- 如果数据库不存在，会自动创建并初始化
- 初始化过程包括创建表结构和默认数据

### 3.2 默认数据
- **角色**：超级管理员、堂区管理员、操作员
- **权限**：用户管理、角色管理、堂区管理、家庭管理、成员管理
- **用户**：默认超级管理员账号
- **村**：默认村（代码：001）

### 3.3 数据库文件位置
- 开发环境：`data/household.db`
- 打包后：与可执行文件同目录的 `data/household.db`

## 4. 登录信息

### 4.1 默认账号
- **用户名**：admin
- **密码**：admin123
- **角色**：超级管理员（拥有所有权限）

### 4.2 权限说明
- **超级管理员**：拥有所有权限，可以管理所有村的数据
- **堂区管理员**：只能管理本村的家庭和成员数据
- **操作员**：只能操作指定范围内的成员数据

## 5. 常见问题及解决方案

### 5.1 启动失败
- **问题**：找不到模块或依赖
  **解决方案**：确保已安装所有依赖，执行 `pip install -r requirements.txt`

- **问题**：数据库初始化失败
  **解决方案**：检查文件权限，确保程序有写入权限

### 5.2 登录问题
- **问题**：用户名或密码错误
  **解决方案**：使用默认账号 `admin` 和密码 `admin123`

- **问题**：登录后无权限访问某些功能
  **解决方案**：检查用户角色和权限设置

### 5.3 数据库问题
- **问题**：数据库文件损坏
  **解决方案**：删除 `data/household.db` 文件，重新启动系统自动初始化

- **问题**：数据丢失
  **解决方案**：定期备份 `data/household.db` 文件

### 5.4 其他问题
- **问题**：界面显示异常
  **解决方案**：确保系统分辨率和显示设置正常

- **问题**：程序崩溃
  **解决方案**：检查系统日志，或重新安装依赖
